WHAT WE FOUND

SQUARESPACE 7.1
🖼️
Hero Image Loaded via Squarespace CDN — No WebP/AVIF
The logo/hero image loads as JPEG at 1500w from squarespace-cdn.com. No next-gen format, no lazy loading hint, and it appears twice in the DOM (mobile + desktop duplicate). This is your #1 LCP killer on mobile — likely adding 800ms–1.5s to load time.
HIGH IMPACT
📋
Duplicate Navigation — Rendered Twice in HTML
The entire navigation menu (30+ links) is duplicated in the HTML source — once for desktop, once for mobile. This bloats the DOM, confuses crawlers, and Squarespace's template system generates both simultaneously. No control to fix without leaving the platform.
HIGH IMPACT
🗺️
Zero Schema Markup (LocalBusiness, Service, FAQ)
No LocalBusiness, Service, FAQ, or Review structured data found on any page. Competitors with schema get rich results in Google — star ratings, FAQ dropdowns, service prices. You're invisible in rich results. Squarespace makes custom schema injection very difficult.
HIGH IMPACT
📄
Thin Location Pages — 20+ Near-Duplicate City Pages
Pages like /clean-truck-check-hayward, /clean-truck-check-lodi, /clean-truck-check-fairfield etc. appear to have minimal unique content. Google flags these as thin/duplicate and may be suppressing or ignoring them in rankings despite the keyword investment.
HIGH IMPACT
🔗
Inconsistent URL Naming Convention
"New Page" is a live indexed page at /new-page. "Top Google Rated Service in Oakland" loads an .html file outside Squarespace CMS. Mix of dash-separated formats, no clear hierarchy. This signals to Google that the site lacks architectural discipline.
MED IMPACT
⚡
No Booking/CTA Optimization — Booking Page Buried
The Book CARB Testing CTA lives deep in the nav. No sticky CTA, no urgency signals on service pages, no price anchoring with social proof. The OBD page doesn't have a visible "Book Now" button above the fold.
MED IMPACT
📱
Mobile Nav Has 30+ Links in Hamburger Menu
Truck fleet managers on mobile are seeing 30+ links when they open the hamburger. This is a conversion killer. Most customers need: Book Test, Services, Call. Everything else is secondary.
MED IMPACT
🛒
Squarespace Commerce Cart — Unnecessary Overhead
A cart icon shows [0] on every page, loading WooCommerce-equivalent Squarespace commerce scripts. If bookings go through Acuity or manual contact, this is dead weight affecting load time on every page visit.
LOW IMPACT
✅
GOOD: SSL, Mobile Responsive, Clean URLs, Fast Hosting
Squarespace handles SSL auto-renewal, is fully mobile-responsive, generates an auto-sitemap, and uses Fastly CDN. These are genuinely good fundamentals and shouldn't be undervalued when comparing platforms.

301 REDIRECT STRATEGY IS EVERYTHING
Every current URL must have a 1:1 301 redirect set before DNS switch. With 30+ location pages indexed and potentially ranked, losing those URLs without redirects would be a Google disaster. This is manageable but requires a redirect map doc before migration starts.
⚡ THE VERDICT — HYBRID APPROACH WINS
Don't nuke and rebuild. The Squarespace site has real indexed pages, some SEO equity, and you can update it yourself — that matters. But Squarespace is capping your ceiling: no schema, no speed control, no programmatic pages at scale, limited schema injection.

Short-term (0–60 days): Fix what you can inside Squarespace NOW — schema injection via code block, compress/replace hero image, consolidate thin location pages, add sticky CTA. These are free wins.

Medium-term (60–180 days): Build a WordPress or custom HTML parallel site with proper schema, speed-optimized, and programmatic city pages. Migrate with a full 301 redirect map. Expected gain: +20–40% organic traffic from schema rich results alone.

The custom HTML argument: For a service business with ~30 static pages, a custom HTML/CSS site on Cloudflare Pages is genuinely superior to both options — 95-100 PageSpeed, $0/month hosting, zero plugin bloat, total schema control. The downside is it requires a developer (or Claude) to update content. Hybrid: WordPress for the blog/CMS, static HTML for service landing pages.
